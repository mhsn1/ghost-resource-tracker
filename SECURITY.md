# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.0.x   | ✅ Yes    |

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Email: security@YOUR_DOMAIN (or use GitHub's private vulnerability reporting)

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond within 48 hours and aim to release a patch within 7 days for confirmed issues.

## Scope

This tool is **defensive only**. Any use of this codebase to build offensive tools, malware, or surveillance software violates the MIT license terms and this project's intent.
